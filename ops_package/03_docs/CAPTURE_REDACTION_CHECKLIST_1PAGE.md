# 캡처 마스킹 체크리스트 1페이지

이 문서는 **운영 리허설 캡처 제출 전 필수 보안 점검**을 위해 사용합니다.

---

## 1) 목적

- 캡처 증빙에서 **비밀값/내부정보 노출 사고를 예방**
- 제출 전 **반드시 1회 확인**

---

## 2) 캡처에 절대 포함 금지 (체크박스)

- [ ] `.env` 파일 내용 또는 경로
- [ ] 키/토큰/비밀값(예: `MASTER_KEY`, `API_KEY`, `TOKEN`, `SECRET`, `PASSWORD`)
- [ ] 인증/서명 값(예: `Authorization`, `X-Signature`, `nonce`, `deviceKeyId`)
- [ ] 내부 IP / 호스트명 / VPN / SSID
- [ ] Windows 사용자명 / PC 이름 / 개인 식별 경로
- [ ] 서비스 계정/권한 설정 화면의 계정 식별정보
- [ ] 로그 원문(payload) 중 민감 필드

---

## 3) 안전한 캡처 원칙(대체 캡처)

- health는 **StatusCode 200만** 보이게 캡처
- Ticket-17.2는 **[PASS]/[FAIL] 근거 라인만** 캡처
- ZIP 생성은 **파일명 목록만** 캡처
- selfcheck는 **PASS/FAIL 요약만** 캡처

---

## 4) 마스킹 방법(초보자용)

1) **영역 캡처(크롭) 우선**
2) 불가피하면 **Paint로 검정 박스 마스킹**
3) 저장 전 **이미지 미리보기로 노출 여부 재확인**

---

## 5) 제출 전 최종 점검(3줄)

1) 캡처 폴더 전체 빠르게 훑기
2) 비밀값이 보이면 **즉시 폐기 후 재캡처**
3) 번들 ZIP 생성 전 **마스킹 완료 체크**
